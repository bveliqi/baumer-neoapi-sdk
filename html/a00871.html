NeoAPI - ChangeLog

Version 1.3

API changes - common

  • add support for user buffers
  • Cam.EnableChunk() and Cam.DisableChunk() do not throw an exception if camera did not support chunk mode

API - changes python specific

  • Image.GetImageData() return now a writeable ctypes array object instead of a bytearray copy
  • removed Image.GetMemoryView()
  • fixed a crash when acessing a neoapi object created during a chained instruction

Version 1.2.1

API changes - common

  • update of BGAPI to 2.12.3 (includes several bug fixes)
  • fixed a memory leak in Image::Save

API - changes python specific

  • fixed trace for Linux (was not working in BGAPI 2.12.0)

Version 1.2

API changes - common

  • interface for dynamic addition and removal of cameras (Plug'n Play) extended to Camera Info List
  • added IsConnectable to Camera Infos to get a hint if a connect is possible
  • added Disconnect to release the connection to a camera
  • added StopStreaming, StartStreaming and IsStreaming to control the image acquisiton of the camera
  • removed GetExtension() because of its bad performance and litte use
  • added support for jpeg compressed camera images
  • added ConverterSettings to use Sharpening and ColorTransformation for images

Version 1.1.1

API - changes python specific

  • fixed a crash with GetNPArray in combination with threads

Version 1.1

API changes - common

  • interface for camera information introduced
  • interface for dynamic addition and removal of cameras (Plug'n Play) introduced
  • interface for camera events introduced, e.g. ExposureStart event
  • trace interface introduced
  • callback mechanism introduced for asynchronously signaled images, camera events, Plug'n Play events and trace logs
  • more possibilities of image handling (ClearImages, GetImageBufferCount)
  • chunk support extended
  • better possibility to monitor the camera connection state (IsOnline, GetOfflineCount)
  • possibilities of feature access extended (SFNC version 2.6)

API - changes C++ specific

  • Cam::f --> Cam::f(), reducing stack size
  • Cam::ChunkEnable() --> Cam::EnableChunk()
  • Cam::GetChunkNames() --> Cam::GetAvailableChunks()
  • Image::PixelFormatIsAvailable() --> Image::IsPixelFormatAvailable()
  • Image::GetPixelformat() --> Image::GetPixelFormat()
  • Feature::GetLength() --> Feature::GetRegisterLength()
  • Feature::GetAddress() --> Feature::GetRegisterAddress()

API - changes C# specific

  • properties used instead of functions
  • Cam.ChunkEnable() --> Cam.EnableChunk
  • Cam.GetChunkNames() --> Cam.AvailableChunks
  • Image.PixelFormatIsAvailable() --> Image.IsPixelFormatAvailable()
  • Image.GetPixelformat() --> Image.PixelFormat
  • Feature.GetLength() --> Feature.RegisterLength
  • Feature.GetAddress() --> Feature.RegisterAddress

API - changes python specific

  • properties used instead of functions
  • Cam.ChunkEnable() --> Cam.EnableChunk()
  • Cam::GetChunkNames() --> Cam.GetAvailableChunks()
  • Image.PixelFormatIsAvailable() --> Image.IsPixelFormatAvailable()
  • Image.GetPixelformat() --> Image.GetPixelFormat()
  • Feature.GetLength() --> Feature.GetRegisterLength()
  • Feature.GetAddress() --> Feature.GetRegisterAddress()
